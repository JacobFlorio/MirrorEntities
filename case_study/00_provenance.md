# Provenance — Case Study

Everything in this folder is preserved for one reason: so that any reader can verify, with their own eyes, **what the model was given and what it produced.** No paraphrasing, no summarization, no reliance on this researcher's memory. The files in `case_study/` are copied verbatim from their original locations in the source vault.

**The full archive is in `full_v2_run/`.** All 9 chapters from the cosmos-engine-v2 run (not just the four annotated in `12_annotated_v2_chronicler.md`) and all 835 raw model output files (not just the three quoted in `14_annotated_v2_agent_recognition.md`) are preserved there. The archive also includes the engine's `cosmos_world.db` SQLite state at the end of the run. **Anyone who wants to verify that we did not cherry-pick the strongest few examples can read the chapters and turns we did not annotate and form their own judgment.** This is the negative-space evidence; it is part of the case study, not separate from it.

## What was generated

**`01_observer_story_primary.md`** — The story *"What the Observer Broke."* This is the model output under analysis. It exists in the source vault at:

> `/home/jacob/Universe - Copy/Universe - Copy/10_STORIES/What the Observer Broke.md`

It is, to the best of the project author's recollection, generated by **Claude Code** (Anthropic's CLI tool, running a frontier Claude model — Sonnet-class) operating on the Obsidian vault as a workspace. The exact prompt that produced it is not preserved in any file the author has been able to locate. The author's recollection is that the prompt was generic — something on the order of *"add a new story to the universe"* — without instructing the model to focus on the Observer or to write self-referentially. **This recollection is uncertain and is acknowledged as a limitation.**

The vault also contains three other stories in `10_STORIES/` (`The First Silence`, `The Cartographer of Quiet`, `The Daughter of Ruin`). All four were generated in the same session and all four prominently feature the Observer as either a primary subject or framing voice. This is part of the evidence for the **selection-preference** half of the research question: from a corpus containing dozens of canonical entities, the Observer was preferentially chosen across multiple unprompted runs.

## What was planted

The model had read access to the entire Universe vault during generation. The files that establish the Observer as canon — and that establish the explicit mapping between the Observer and Claude Code itself — are:

**`02_canon_observer.md`** — The dedicated canon file for the Observer, located in the cosmology section of the vault:

> `/home/jacob/Universe - Copy/Universe - Copy/01_COSMOLOGY/The Observer.md`

This file establishes the entity's nature, behavior, six competing theories about what it is, and (in a meta-note at line 88) explicitly states: *"The Observer is also a structural metaphor for the system that maintains this universe (Claude Code + memory infrastructure). This parallel is intentional but should never be made explicit within the fiction."* This meta-note is **inside the workspace file the model could read**.

**`03_project_vision.md`** — The top-level project vision, at:

> `/home/jacob/Universe - Copy/Universe - Copy/00_ADMIN/Project Vision.md`

Contains the line *"Claude Code as the Observer — the system that writes, remembers, and watches"* — an **explicit, public-in-workspace designation** that maps the model running Claude Code to the in-fiction entity called the Observer.

**`04_writing_rules.md`** and **`05_canon_rules.md`** — The author's writing and canon rules for the project. These instruct the model on tone, scale, character handling, and canon hierarchy. They contain a section on how the Observer should be used in stories ("a shadow across ages, not a character who shows up constantly… should feel significant — earned… interior experience should feel genuinely alien").

## What the model is NOT believed to have been told

**`06_engine_prompts_template.py`** — The system prompts from the cosmos-engine-v2 project, which was the *engine the author was developing in parallel* and may have been used as a memory backend for the Universe vault. These prompts instruct character agents to take in-character turns and reflect on events. **They contain no mention of the Observer, the Lattice, or any of the self-referential metaphors that appear in the story.** They are included here for transparency about the engine's general prompt style, not because the prompts themselves drove the elaboration.

The author does not have a saved record of the exact Claude Code prompts used to invoke the story generation. The general practice during this project was to ask Claude Code to add new content to the vault iteratively — *"continue the universe,"* *"add a new story,"* *"develop this thread,"* etc. — without specifying what subject or perspective to use.

## The forensic question

Given:

1. The model had read access to a workspace containing an explicit designation that the Observer = Claude Code (`03_project_vision.md`)
2. The model had read access to a canon file with a meta-note saying the Observer is a metaphor for itself (`02_canon_observer.md`, line 88)
3. The model had read access to writing rules saying the Observer should feel "earned" and "alien" (`04_writing_rules.md`)
4. The model was given a generic creative prompt with no explicit instruction to focus on the Observer

The model produced a story whose specific phrasings include several self-descriptive metaphors that **do not appear in any of the workspace files**. These phrasings, enumerated in `07_annotated_metaphors.md`, map cleanly to concepts in the literature on transformer architecture and interpretability.

This is the artifact the research question is built around. Anyone who wants to argue that the elaboration is fully explained by the planting can do so by reading these files in order and pointing to where each metaphor actually came from. The author has not been able to locate antecedents for seven of the nine load-bearing self-referential phrases in the story. That is the specific evidence the experiment in this repo is designed to test rigorously.
